## <b>AI IDE BAS</b> 能做什么？
AI IDE BAS 帮助 <b>生成和校验各类产物</b>。工作按角色组织 — 每个角色有各自的产物。

---

## 主要角色：
### <b>业务分析师</b>
- 创建 User Stories；
- 创建 Use Cases；
- 创建 PlantUML 格式的业务流程活动图；
- 创建验收标准（Acceptance Criteria）；
- 编制项目术语表；
- 收集项目干系人信息；
- 校验所选业务分析师产物。
### <b>系统分析师</b>
- 创建 ERD（实体关系图）；
- 创建数据模型；
- 创建时序图；
- 创建 OpenAPI 规范；
- 创建 Kafka Message Broker 的 AsyncAPI 规范；
- 创建非功能需求；
- 校验所选系统分析师产物。
### <b>解决方案架构师</b>
- 创建上下文图；
- 创建容器图；
- 创建组件图；
- 校验所选解决方案架构师产物。
### <b>Reviewer</b>：
- 校验项目在需求与业务/系统分析师产物方面的质量；
- 按网络安全要求校验项目；
- 校验架构决策；
- 评估方案成本；
- 从支持工程角度校验项目。

---

## 附加角色：
### <b>Helper</b>
- 提供 AI IDE BAS 及扩展使用的咨询与提示。
### <b>设计师</b>
- 创建可点击原型。
### <b>项目经理</b>
- 创建冲刺待办的文字描述；
- 创建开发任务说明；
- PlantUML 格式甘特图；
- PlantUML 格式依赖图。
