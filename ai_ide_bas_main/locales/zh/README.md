## <b>AI IDE BAS</b> 能做什么？
按角色生成产物。
---

## 主要角色：
### <b>业务分析师</b>
- 创建用户故事（User Stories）
- 创建用例（Use Cases）
- 创建 PlantUML 格式的业务流程活动图
- 创建验收标准（Acceptance Criteria）
- 形成项目术语表
- 收集项目干系人信息
- 复核业务分析师产物
### <b>系统分析师</b>
- 描述后端逻辑
- 创建 ERD（实体‑关系图）
- 创建数据模型
- 创建时序图
- 创建 OpenAPI 规范
- 创建 Kafka Message Broker 的 AsyncAPI 规范
- 创建非功能性需求
- 复核系统分析师产物
### <b>解决方案架构师</b>
- 创建上下文图
- 创建组件图
- 复核架构师产物

---

## 其他角色：
### <b>设计师</b>
- 创建可点击原型
### <b>项目经理</b>
- 创建冲刺待办的文本描述
- 创建开发任务说明
- PlantUML 格式的甘特图
- PlantUML 格式的依赖关系图

为检查你的项目提供了单独的角色。
### <b>Reviewer</b>:
- 检查业务与系统分析师需求与产物的质量
- 按网络安全要求检查项目
- 检查项目架构决策
- 从支持工程师视角检查项目

如需更详细的功能咨询，可使用 Helper 角色 =)
